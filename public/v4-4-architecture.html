V4.4 核心升級（學習 obra/superpowers 框架）

V4.2 的最大問題：缺乏 code review 導致品質不穩、任務粒度太粗導致失敗率高、debug 沒有系統方法。V4.4 引入 6 大機制：兩階段 Code Review 確保品質、5-10 分鐘任務粒度 降低風險、自動驗證閘門 取代手動確認、TDD 強制流程 預防 bug、Git Worktree 隔離開發、Systematic Debugging 4 階段修復。

4

TDD 強制流程

RED → GREEN → REFACTOR
RED：先寫失敗的測試。GREEN：寫最少的 code 讓測試通過。REFACTOR：重構但保持測試通過。沒有測試的 code 不允許 commit。

  • bug 在產生的瞬間就被測試捕捉
  • 重構有安全網，不怕改壞
  • 測試即文件，需求永不遺失

Systematic Debugging

4 階段結構化修復
Phase 1 Reproduce：穩定重現問題。Phase 2 Isolate：縮小範圍找到根因。Phase 3 Fix：最小修改修復。Phase 4 Verify：確認修復且無副作用。
