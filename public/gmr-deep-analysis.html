General Motion Retargeting — 人形機器人動作採集應用
📅 2026-04-02 | 🔬 三模型交叉質疑（Grok-4 / GPT-4o-mini / GLM-5）| 📊 Stanford ICRA 2026 + CoRL 2025

📋 結論先講

GMR 是目前開源 retargeting 中品質最好的方案，接近 Unitree 閉源水準。但它是工具鏈的一環，不是完整解決方案。從人類動作到機器人實際執行，GMR 只做了前 20%（retargeting），後面 80%（RL 訓練 + sim-to-real + 部署）才是真正的成本所在。

核心判斷：適合有 RL/simulation 團隊的研究機構或機器人公司；不適合沒有技術團隊的通路商直接導入。

🏗️ 技術架構

📹 人類動作輸入 → 🔍 非均勻局部縮放
(Non-uniform Local Scaling) → ⚙️ 兩階段 IK 優化
(Coarse → Fine) → 🦿 機器人關節軌跡 → 🧠 TWIST RL 訓練
(Teacher-Student) → 🤖 實體機器人部署

GMR 本體只負責前 3 步（縮放 + IK + 輸出軌跡）。要讓機器人真正動起來，還需要後面的 RL 訓練（TWIST）和實體部署。

核心技術亮點

  • 非均勻局部縮放：逐段骨骼獨立縮放，避免等比縮放造成的足部滑動和穿模（PHC/ProtoMotions 的主要問題）
  • 兩階段優化：第一階段粗解全身位置，第二階段 Mink 庫精調關節誤差最小化
  • 速度限制約束：預設啟用 velocity limits (3π rad/s)，防止超出機器人物理極限
  • CPU 即時：不需 GPU，單核 <10ms，適合即時遙操作場景
  • 多格式支援：SMPLX / BVH / FBX / PICO / 單目視頻（GVHMR pipeline）

⚠️ 關鍵缺陷與風險（三模型交叉驗證）

以下問題由 Grok-4（運動學專家）、GPT-4o-mini（ML 工程師）、GLM-5（企業顧問）三個獨立模型分別分析後交叉比對，標記共識度。

問題嚴重度共識詳細說明
Sim-to-Real Gap 嚴重 高 3/3 17 種機器人僅 Unitree G1 有實體驗證。仿真忽略摩擦係數變化、感測器噪聲、馬達延遲。真機 tracking error 常比仿真放大 1.5-3 倍。首次上機成功率可能僅 20-50%。
不處理物體互動 高 3/3 GMR 只做無接觸動作的 retargeting。抓取、避障、工具使用完全不在 scope 內。工業場景需要額外的 MPC/RRT* 規劃器，會破壞 <10ms 即時性。
手部操作極度有限 高 3/3 僅 G1+Hands 支援手部（43 DOF），但 retargeting 品質粗糙。無法做精細操作（捏取、工具握持）。工業場景的靈巧手需求完全無法滿足。
Actuator Dynamics Mismatch 高 2/3 GMR 輸出的是理想關節軌跡，忽略馬達扭矩限制、齒輪間隙、死區、熱衰減。仿真能追的軌跡，真機 controller 未必追得到。Hip/Knee/Ankle torque margin 不足時動作會變形。
需完整 URDF 模型 中 3/3 每種新機器人需自建 URDF/MJCF，包含 link inertia、joint axis、collision geometry、actuator spec。很多商用機器人 CAD 不完整，inertia/friction 參數缺失。自動 URDF 生成工具精度不足。
兩階段 IK 局部最優問題 中 2/3 Hessian 分析（NMR 論文 arxiv 2603.22201）證明此類優化本質非凸。複雜姿勢（蹲起+轉身）容易卡在局部最優，產生關節跳動、自穿模。GMR 的兩階段策略緩解但未根本解決。
RL 訓練成本與不穩定性 高 3/3 TWIST 訓練需要 RTX 4090 跑 1-2 天（理想值）。實際 debug 週期常是 3-10 次實驗，一輪更新 1-3 週。每個新動作需要 fine-tune policy，複雜動作可能需要 1-4 週。RL 策略抖動大，retargeting 輸入變異被放大。
單人維護風險 中 2/3 GMR + TWIST 主要由 Yanjie Ze（Stanford 博士生）一人維護。GitHub commits 少，社群貢獻薄弱。若主要作者畢業或轉向，專案可能停滯。
延遲敏感度 高 2/3 即時遙操作場景下，end-to-end latency 超過 15-30ms 動態動作就開始不穩。control loop 從仿真理想 500-1000Hz 降到真機有效 200-400Hz。balance-critical motion 額外 10ms latency 就可能大幅降低成功率。
非均勻縮放忽略生物力學 中 2/3 純幾何縮放不考慮質量分布、慣量變化。某些 retargeted motion 在動力學上不可執行，需要額外 motion editing / stabilization pass。長期跑可能造成關節過載，損壞硬體。

關鍵發現：NMR (arxiv 2603.22201)

2026 年最新的 Neural Motion Retargeting 論文直接引用 GMR 作為 baseline 並指出其局限：

  • 🎯 通過 Hessian 分析證明 GMR 類 IK 優化本質非凸，容易陷入局部最優
  • 🎯 提出 VAE 聚類 + RL expert 修正 + CNN-Transformer 架構，消除關節跳動
  • 🎯 在武術、舞蹈等動態任務上，NMR 顯著減少自穿模
  • ⚠️ 但 NMR 需要 GPU 推論，不支援 CPU 即時，且僅驗證 Unitree G1

結論：GMR 在幾何優化路線中是最佳選擇，但神經網路路線（NMR）正在追趕甚至超越。

🎯 最終建議

如果目標是「快速展示」

Phase 0 + Phase 1，投入 NT$80-100 萬，3-4 個月做出 1 個實機 demo。風險可控，失敗了就當學費。

如果目標是「建立服務能力」

不建議自己搞。找技術合作夥伴（台大/清大/交大機器人實驗室），和椿負責硬體通路 + 客戶關係，學界負責技術開發。預算 NT$200-300 萬 / 年，2 年內可驗證商業模式。

如果目標是「了解技術趨勢」

最推薦。跑通 Phase 0（2-4 週，成本接近零），了解整個 pipeline。密切關注 NMR（神經 retargeting）和 TWIST2 的進展。GMR 是過渡技術，神經網路路線才是終局。
